    • 批改建議

      HTML

      1. 在 class 的命名上會建議同學採用慣例上的小駝峰命名法 (lower camel case) ，讓第一個單字以小寫字母開始，而第二個單字的首字母大寫來提升程式碼的可讀性

      2. 建議不要使用行內樣式 style 標籤以及 style= "..." 屬性來指定樣式，由於 inline style 的權重較高，樣式較難覆蓋，會提高日後在維護上的難度。建議可新建一個 .css 檔案將整份作業的樣式放置至此並於 HTML 以 link:src 來引入

      3. h1 的部分不需要固定寬度，可在文字內容中需要斷行的部分插入 br 斷行標籤。結構如下所示：h1 Joanne br Chen /h1

      4. 區塊標題的部分建議改為如下所示，能使結構更精簡。h2 class="...">...p class="...">.../p>/h2>

      5. 個人資料的部分 span 標籤可以改為 ul > li 結構，並於需要推出距離的 li 標籤上命名 class，依照設計稿所標示的數值設定 margin-bottom，不需要使用 br> 斷行標籤。像是這樣：

      ul class="...">
      li>陳久安 Joanne Chen/li>
      li>.../li>
      li class="...">.../li>
      li>UI 設計修行中／立志當斜槓青年/li>
      li>.../li>
      li>.../li>
      /ul>

      6. 學歷和技能的部分同第 5 點，結構則會像是這樣：

      ul class="...">
      li>國立六角師範大學 視覺設計學系 學士學位 | 2016-2020/li>
      li>國立六腳高級中學 美術班 | 2013-2016/li>
      /ul>

      和這樣：
      div class="BlueBackGround">
      h3 class="...">...p class="...">.../p>/h3>
      ul class="...">
      li>Adobe Photoshop/li>
      li>Adobe Illustrator/li>
      li>Adobe XD/li>
      li>Adobe Indesign/li>
      li>C4D/li>
      li>Adobe Premiere/li>
      li>語言：中文、英文/li>
      /ul>
      /div>

      7. 工作經驗的部分同樣可以做縮減，讓結構更精簡。可參考老師提供的參考圖：li 內包 h3 img p

      8. 聯絡方式的結構同第 5 及 6 點，另建議可在 Mail 和 Phone 內包上 a 連結，當使用者點擊該區塊時，就會自動開啟相對功能；並可搭配 :hover 效果增加使用者體驗。而三個 icon 的部分也建議改為 ul > li 結構，並於 img 標籤外包上一層 a 連結以及搭配 :hover 效果增加使用者體驗，如下所示：

      footer class="BlueBackGround">
      h3 class="...">...p class="...">.../p>/h3>
      ul>
      li>Mail：a href="mailto:xxxd@hexschool.com">xxxd@hexschool.com/a>/li>
      li>Phone：a href="tel:+0910123456">0910123456/a>/li>
      li>Kaohsiung, Taiwan/li>
      li>作品連結：a href="https://www.behance.net/xxxdddddd">https://www.behance.net/xxxdddddd/a>/li>
      /ul>
      ul>
      li>a href="#">.../a>/li>
      li>a href="#">.../a>/li>
      li>a href="#">.../a>/li>
      /ul>
      /footer>

      CSS

      1. 同學於 HTML 第 141 行使用的是 width: 1200px; 和 margin:47px auto auto auto;，這邊會建議同學將 margin 的設定改為 margin: 0 auto 來達到水平置中，上方需要推出距離的部分可使用 padding-top 來達成

      2. 請盡量使用 class 選擇器來指定樣式，由於一個網站內可能會有數個不同用途的 h2, h3, p 標籤等等，可避免單純使用類別選擇器時可能產生樣式之間彼此衝突，造成程式碼可讀性降低的狀況

      3. h1 的部分不需要固定高度，可依據設計稿標示來設定行高 line-height 以撐出高度

      4. 字型 font-family 的部分建議在 body {…} 內統一設定，中英文需要分別設定不同字體的部分可以將其改寫為 font-family: 'Roboto', 'Noto Sans';，讓英文字體不支援的中文字可以套用到下一個順位的字體上

      5. 線條的部分不需要使用 hr> 水平分隔線標籤，可在需要添加分隔線的元素上使用 border-bottom 來調整樣式

      6. 請不要使用 p style="height: ...;">/p> 來撐出高度，可在元素上設定 margin-bottom 來推出距離

      7. .BlueBackGround 區塊的部分不需要設置 min-height，技能內容的部分也請不要設置 margin: 0 840px;，建議以固定外層容器寬度和搭配 margin: 0 auto 的方式來調整文字內容

      8. text-align 的設定請根據當下需要調整的元素更改即可，同學目前所使用的 .BlueBackGround>div>*, 標題標籤以及行內樣式分別設定的方式會使覆蓋樣式的權重難以判讀
